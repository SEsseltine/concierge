# Concierge - Product Specification

Concierge is a web app that makes trip planning, organization, and management simple. Users can track expenses in multiple currencies, split expenses with other travellers, and map out their trip.

---

## Technical Stack

| Layer | Technology |
|---|---|
| Frontend framework | SvelteKit |
| Frontend hosting | Cloudflare Pages |
| Backend / database | Supabase (Postgres, Auth, Storage, Realtime) |
| Icons | Lucide |
| Maps (embedded) | Leaflet + OpenStreetMap |
| Geocoding / place search | Nominatim (OpenStreetMap) |
| Directions | Deep links — `maps://` (iOS), `geo://` (Android), Google Maps (desktop fallback) |
| Currency exchange rates | Frankfurter API |
| Push notifications | Web Push API (service worker); SMS fallback via Supabase Edge Functions + Twilio |

## Product Requirements

### Authentication

As a user, I can...

1. Sign up with email & password
2. Sign up with phone number
3. Sign in with email & password
4. Sign in with phone number
5. Sign in with a passkey
6. Sign in with a magic link (email)
7. Recover a forgotten password via email

---

### Trip Management

As a user, I can...

1. Create a trip with a name, destination, start date, and end date
2. Edit a trip's name, destination, or dates
3. Delete a trip I own
4. View all my trips (upcoming, active, past)
5. Invite a Collaborator to a trip by email or phone number
6. Invite a Viewer to a trip by email or phone number
7. Change a trip member's role (Collaborator ↔ Viewer)
8. Remove a member from a trip (owner or collaborator only)
9. Leave a trip I was invited to

#### Roles & Permissions

| Action | Owner | Collaborator | Viewer |
|---|---|---|---|
| View itinerary & expenses | ✅ | ✅ | ✅ |
| Add / edit / delete activities | ✅ | ✅ | ❌ |
| Add / edit / delete expenses | ✅ | ✅ | ❌ |
| Invite members | ✅ | ✅ | ❌ |
| Remove members / change roles | ✅ | ❌ | ❌ |
| Delete trip | ✅ | ❌ | ❌ |
| Leave trip | — | ✅ | ✅ |

### Itinerary

As a user, I can...

1. Add an activity to a trip (name, type, date/time, location, notes)
   - Activity types: Flight, Accommodation, Transport, Food & Drink, Attraction, Other
2. Search for a location by name when adding or editing an activity (via Nominatim)
3. Edit an activity
4. Delete an activity
5. Reorder activities within a day
6. View the full trip itinerary in chronological order
7. View the itinerary filtered by day

---

### Expenses

As a user, I can...

1. Add an expense to a trip (description, amount, currency, category, date, paid by)
   - Categories: Accommodation, Transport, Food & Drink, Activities, Other
2. Edit an expense
3. Delete an expense
4. Record an expense in any fiat currency; amounts are displayed converted to the trip's base currency (via Frankfurter API)
5. Set the base currency for a trip
6. View a running total cost for the trip in the base currency
7. View all expenses broken down by category
8. View a cost summary split by person (who owes what to whom)
9. Mark a debt as settled between two travellers
10. View settlement history for a trip

---

### Map

As a user, I can...

1. View all trip activities that have a location pinned on an embedded map (Leaflet + OpenStreetMap)
2. Tap or click a pin to see activity details
3. Toggle pins grouped by day
4. Tap "Get Directions" on an activity to open the device's native maps app (or Google Maps on desktop)

---

### Notifications

As a user, I can...

1. Receive a push notification (Web Push / PWA) when:
   - I am invited to a trip
   - A collaborator adds, edits, or deletes an activity on a trip I'm on
   - A new expense is added to a trip I'm on
   - A debt is marked as settled
2. Receive an SMS notification for the above events if push is unavailable or not opted into (requires phone number on account)
3. Receive an email notification for the above events as a final fallback (requires email on account)
4. Configure my notification preferences per channel (push, SMS, email) and per event type

**Delivery priority:** Push → SMS → Email, based on what the user has set up and opted into.

## Non-Functional Requirements

- **Platform:** Web (responsive, mobile-first), installable as a PWA on iOS and Android
- **PWA:** App ships with a `manifest.json` and service worker; users are prompted to install after creating their first trip
- **Push notifications:** Delivered via Web Push API; requires Safari 16.4+ on iOS (known limitation for older devices); falls back to SMS then email
- **Offline:** Itinerary view is readable offline via service worker caching; expense edits and activity changes queue locally and sync when reconnected
- **Performance:** Core pages load in under 2 seconds on a standard mobile connection
- **Currency rates:** Frankfurter API rates refresh at least once daily; historical rates are used for past expenses
- **Localisation:** Dates and times respect the user's locale; metric and imperial supported where relevant

## Out of Scope (v1)

- Native iOS / Android apps
- Booking or purchasing activities directly in-app
- AI-generated itinerary suggestions
- Group chat / messaging between travellers
- Offline expense entry (expenses require a connection to sync exchange rates)
